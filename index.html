Approach

Collect. Analyse. Deliver.

Asking questions matters.

Clean data doesn't tell you what to look for. Understanding the problem first — before writing a single line of code — is the difference between a good model and a useful one.

Clarity over complexity.

An insight buried in a 50-tab dashboard isn't an insight. I build reports and models that communicate — to technical teams and stakeholders alike.

Build for change.

Data pipelines break. Business requirements shift. I write code and workflows that expect this — modular, documented, and easy to extend.
